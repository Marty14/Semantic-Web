🛡️ CyberSecOnto - Documentazione del ProgettoCyberSecOnto è un'ontologia OWL nel dominio della cybersecurity che modella le relazioni tra sistemi informatici, vulnerabilità, attacchi e mitigazioni. Progetto realizzato per il corso di Web Reasoning (Prof.ssa Marianna Nicolosi Asmundo), Università degli Studi di Catania.  📋 IndicePanoramicaFile dell'OntologiaScript PythonCome Usare il ProgettoStruttura delle DirectoryPanoramicaIl progetto utilizza un ecosistema di tecnologie integrate per l'automazione dell'analisi del rischio:  OWL per la definizione di un dominio espanso a 58 classi e proprietà logiche avanzate (transitive, simmetriche, asimmetriche e inverse).  Description Logics per validare la coerenza tramite restrizioni universali, esistenziali, di cardinalità e operatori booleani complessi.  SWRL per le regole di inferenza e propagazione del rischio matematico (CVSS).  SPARQL per interrogare dinamicamente la base di conoscenza.  Python + owlready2 per il popolamento procedurale di scenari Enterprise e testing automatizzato.  File dell'OntologiaUntitled_2.rdfTipo: Ontologia OWL base (formato RDF/XML)Descrizione: Questo è il file strutturale principale dell'ontologia creato inizialmente con Protégé (T-Box). Definisce la gerarchia delle classi, le object/data properties e le restrizioni logiche.cyberseconto_populated_2.owlTipo: Ontologia popolata con dati di test EnterpriseDescrizione: Versione dell'ontologia generata dall'esecuzione dello script di popolamento. Contiene istanze realistiche di ThreatActors, InformationAssets (PII, FinancialData) e un'articolata rete di difese e server[cite: 6].cyberseconto_inferred.owlTipo: Ontologia con inferenze materializzateGenerato da: apply_swrl_rules.pyDescrizione: Il file finale e completo. Racchiude la totalità della base di conoscenza, consolidando sia i dati popolati che le deduzioni logiche estratte dai Reasoner.  Script Pythonpopolamento.pyScopo: Popolare l'ontologia per simulare una vera infrastruttura IT.Cosa fa:Carica la struttura base dell'ontologia.Genera un ecosistema aziendale mappando server di produzione, workstation e dispositivi di rete[cite: 6].Istituisce attori di minaccia specifici (es. Hacker, OrganizedCrime) e attacchi mirati (es. Ransomware, DDoS)[cite: 6].Associa asset informativi critici (dati personali e finanziari) ai rispettivi database[cite: 6].Implementa l'architettura di difesa assegnando firewall e crittografia[cite: 6].Salva il grafo generato in cyberseconto_populated_2.owl.apply_swrl_rules.pyScopo: Applicare programmaticamente le regole SWRL fungendo da motore inferenziale ibrido.  Cosa fa:Carica l'ontologia popolata.Applica Regola 1 (CriticalVulnerability): Classifica le vulnerabilità con CVSS $\ge$ 8.0.Applica Regola 2 (HighRiskSystem): Propaga la criticità identificando i sistemi compromessi.Salva i risultati resi persistenti in cyberseconto_inferred.owl.  Come usarlo:Bashpython3 apply_swrl_rules.py
test_sparql.pyScopo: Eseguire interrogazioni avanzate per l'estrazione automatizzata dei report[cite: 3].Cosa fa: Esegue la suite completa di 6 query SPARQL sul modello inferito:Query 1 - Sistemi Critici e loro Vulnerabilità (CVSS > 7.0)[cite: 3].Query 2 - La Catena d'Attacco Completa (Mappatura Attore $\rightarrow$ Attacco $\rightarrow$ Bersaglio $\rightarrow$ Vulnerabilità)[cite: 3].Query 3 - Dati Sensibili a Rischio (Information Assets in pericolo)[cite: 3].Query 4 - Architettura di Difesa (Mappatura Mitigazioni $\rightarrow$ Sistemi protetti)[cite: 3].Query 5 - Mappa delle Comunicazioni tra Server (sfruttando le proprietà simmetriche)[cite: 3].Query 6 - Statistiche Vulnerabilità per Severità (Conteggio aggregato Critical/High/Medium)[cite: 3].Come usarlo:Bashpython3 test_sparql.py
Script di Debug (debug_onto.py e debug_properties.py)Scopo: Ispezionare programmaticamente le entità dell'ontologia durante la fase di sviluppo.debug_onto.py: Elenca l'albero completo delle classi e gli individui istanziati.debug_properties.py: Elenca le Object Properties e Data Properties attive.Come Usare il ProgettoSetup InizialeInstalla le dipendenze necessarie per l'ambiente Python:Bashpip3 install owlready2
(Opzionale) Verifica la presenza di Java nel sistema per il supporto ai Reasoner tradizionali:Bashjava -version
